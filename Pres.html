QuickNote

Crafting Concise Narratives through RAG-Driven Document Summarization

"This research explores how Retrieval-Augmented Generation (RAG) can revolutionize document summarization by combining external knowledge retrieval with generative AI, producing accurate, contextually relevant summaries while minimizing hallucinations."

Problem Statement

Traditional Large Language Models (LLMs) suffer from critical limitations: generation of hallucinated data, high inference costs, lack of explainability, and inability to incorporate real-time knowledge. These fully-parametric approaches contain all knowledge within model parameters, making updates expensive and labor-intensive.

Literature Review

Existing solutions include GPT-5, Gemini, and LLaMA models which excel at text generation but struggle with factual accuracy in rapidly evolving domains like healthcare, finance, and scientific research.

Research Gap / Innovation

RAG bridges the gap by combining retrieval from external knowledge bases with generative models, enabling dynamic, up-to-date responses without costly retraining. This hybrid approach enhances factual accuracy and explainability.

RAG Architecture

Retriever

Dense Passage Retrieval (DPR) encodes queries and searches for semantically similar documents using vector similarity

Generator

Transformer-based models like BART, T5, or LLaMA generate responses by reasoning over retrieved context

Augmentation

Combines retrieved data with query representation to create enriched context for generation

System Methodology

Workflow Overview

1. File Upload & Validation

Flask-based REST API accepts PDF uploads, validates format, generates unique session IDs for user isolation, and stores files temporarily for processing.

2. Text Extraction & Chunking

PyPDFLoader extracts text from PDFs. RecursiveCharacterTextSplitter divides content into ~5000 character chunks with 800 character overlap for context preservation.

3. Session Management

In-memory storage maintains user sessions with extracted chunks, metadata, and conversation history without requiring external databases.

4. Query Processing & Generation

User queries retrieve relevant chunks, create enriched prompts, and send to LLaMA-3.1-8B via Groq API for contextualized response generation.

Dataset / Input

Tested with 50 academic PDFs (80-120 pages), educational and research datasets including technical papers and documentation

Model / Architecture

LLaMA-3.1-8B via Groq API - resource-efficient model optimized for document-grounded QA with low latency

Implementation

Flask + PyPDF + Groq API

Key Innovations

Lightweight Architecture

No vector database required - in-memory chunk-based retrieval for fast processing of small-to-medium documents

Context Preservation

Overlapping chunks ensure semantic continuity across document sections

Efficient Inference

Groq platform enables 190-240ms latency for responsive real-time interactions

Session Isolation

Unique session IDs prevent data contamination between concurrent users

Results & Analysis

Overall Accuracy 78%
Localized Queries (Definitions) 88%
Average Latency 215ms

Tested on 50 PDFs with 100 manually prepared questions. System excelled at localized information (88% accuracy) but showed limitations with conceptually dispersed documents (63% accuracy), indicating areas for improvement in retrieval strategies.

RAG vs Fine-Tuning

RAG Approach

  • Dynamic knowledge retrieval without retraining
  • Always up-to-date with external data sources
  • Verifiable and explainable outputs
  • Lower computational costs
  • Reduces hallucinations through factual grounding

Fine-Tuning Limitations

  • Static knowledge frozen in parameters
  • Requires complete retraining for updates
  • High computational and time costs
  • Risk of catastrophic forgetting
  • Model staleness with outdated knowledge
